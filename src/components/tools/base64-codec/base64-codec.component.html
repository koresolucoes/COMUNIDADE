A criptografia com senha é uma forma poderosa de proteger dados sensíveis que você precisa armazenar fora do n8n, como em um banco de dados ou numa planilha Google Sheets.

Caso de Uso: Proteger uma Chave de API no Google Sheets Imagine que você precisa usar uma chave de API, mas não quer deixá-la em texto puro numa planilha. Você pode usar esta ferramenta para criptografá-la e depois descriptografá-la com segurança dentro do seu workflow n8n.

Passo a Passo:

  1. Criptografe a Chave: Use esta ferramenta para criptografar sua chave de API. Insira a chave no campo "Entrada", digite uma senha forte e clique em "Criptografar". Copie o resultado da "Saída".
  2. Armazene com Segurança: Cole o resultado criptografado em sua planilha Google Sheets.
  3. Gerencie a Senha no n8n: NUNCA coloque a senha diretamente no seu código. Vá em "Credentials" no n8n, crie uma "Header Auth" e coloque sua senha no campo "Value". Dê um nome, por exemplo, `minha-senha-secreta`.
  4. Leia os Dados no Workflow: Use o nó "Google Sheets" para ler a chave de API criptografada da sua planilha.
  5. Descriptografe no n8n: Adicione um nó "Python". Copie o código da aba Python.
    • Dependências: Garanta que a variável de ambiente N8N_PYTHON_PACKAGES na sua instância n8n inclua cryptography.
    • Entrada (Senha): Para usar a senha das credenciais, adicione uma expressão: Clique em "Add Expression" > "Current Node" > "Input Data" > "Credentials" > "Header Auth". Selecione a credencial que você criou. No código Python, substitua o valor de `password` por `items[0].json['credentials']['value']`.
    • Entrada (Dados): Substitua `encrypted_data_b64` pelo valor vindo do nó do Google Sheets, por exemplo: `items[0].json['suaColuna']`.
    • Saída: O script retornará um item com um campo JSON chamado decryptedResult.
  6. Use a Chave: No próximo nó (ex: "HTTP Request"), use a chave descriptografada com uma expressão como {{ '{{' }} $('Python').item.json.decryptedResult {{ '}}' }}.
Aviso de Segurança: A abordagem com o nó "Execute Code" (JavaScript) também funciona, mas o nó Python é geralmente mais seguro e recomendado em ambientes n8n, pois evita problemas de desativação do módulo `crypto` do Node.js.

A codificação Base64 é essencial para transmitir dados binários (como arquivos) ou para autenticação.

Codificando Arquivos Binários

Quando um nó como HTTP Request baixa um arquivo, os dados ficam disponíveis em $binary.data. Para enviá-lo para uma API que espera Base64, use um nó Move Binary Data:

  1. Defina Mode como "To File".
  2. Ative a opção Destination is Base64.
  3. No campo Destination Property Name, digite o nome do campo que conterá a string Base64 (ex: `fileContent`).

Alternativamente, no nó Code (JS): item.json.fileContent = $binary.data.toBase64();

Decodificando para Arquivo Binário

Se você tem uma string Base64 e quer usá-la como um arquivo (ex: para fazer upload), use o nó Move Binary Data:

  1. Defina Mode como "From JSON".
  2. Ative a opção Data is Base64.
  3. No campo Source Property Name, referencie o campo com sua string Base64 (ex: {{ '{{' }} $json.fileContent {{ '}}' }}).

O nó seguinte agora terá acesso aos dados binários em $binary.data, pronto para ser usado.
