A Linguagem da Web: HTTP e APIs

Entenda como a web se comunica através de requisições HTTP e o que são APIs (Interfaces de Programação de Aplicativos).

No artigo anterior, vimos que a internet funciona como uma conversa entre um cliente (seu navegador) e um servidor. Agora, vamos aprender o "idioma" dessa conversa: o HTTP, e como o cliente sabe quais "pedidos" pode fazer ao servidor, através de uma API.

O que é HTTP? O Garçom Digital

Continuando nossa analogia do restaurante, o HTTP (HyperText Transfer Protocol) é o garçom. Ele é o protocolo, o conjunto de regras que define como os pedidos (requisições) e as entregas (respostas) devem ser formatados para que o cliente e a cozinha se entendam perfeitamente.

Toda comunicação HTTP tem dois lados:

  • Requisição (Request): O cliente pedindo algo ao servidor.
  • Resposta (Response): O servidor entregando o que foi pedido (ou explicando por que não pôde).

Os Verbos do HTTP: O Que Você Quer Fazer?

Uma requisição HTTP sempre vem com um "verbo" que indica a intenção do pedido. Os 4 principais são:

GET

"Me traga algo." É usado para ler ou buscar dados. Quando você acessa um site, seu navegador faz uma requisição GET.

POST

"Anote este novo pedido." É usado para criar um novo dado. Quando você envia um formulário de cadastro, está fazendo um POST.

PUT / PATCH

"Corrija meu pedido." É usado para atualizar um dado existente. PUT substitui o dado inteiro, enquanto PATCH atualiza apenas uma parte.

DELETE

"Cancele meu pedido." É usado para remover um dado.

Códigos de Status: A Resposta do Garçom

Depois de fazer um pedido, o garçom sempre te dá uma resposta. No mundo HTTP, essa resposta vem na forma de um número de 3 dígitos chamado Código de Status. Os mais comuns são:

  • 200 OK: "Pedido recebido e entregue com sucesso!"
  • 301 Moved Permanently: "O que você pediu mudou de lugar. Vá para este novo endereço."
  • 404 Not Found: "Procurei por todo lado, mas não encontrei o que você pediu."
  • 500 Internal Server Error: "Deu um problema na cozinha! Não consegui preparar seu pedido."

O que é uma API? O Cardápio do Restaurante

Ok, você sabe como pedir (HTTP), mas o que você pode pedir? Você não pode chegar em uma pizzaria e pedir sushi. Você precisa de um cardápio.

Uma API (Application Programming Interface) é o cardápio digital do servidor. Ela é um "contrato" que define exatamente quais requisições um servidor aceita, que informações você precisa enviar e o que você receberá de volta.

Uma API REST é o tipo mais comum na web. Ela define "endpoints" (os pratos do cardápio) que combinam um verbo HTTP com um caminho (URL).

Exemplo: Cardápio de uma API de Tarefas

  • GET /tasks → "Me traga a lista de todas as tarefas."
  • GET /tasks/123 → "Me traga os detalhes da tarefa com ID 123."
  • POST /tasks → "Crie uma nova tarefa com os dados que estou enviando."
  • DELETE /tasks/123 → "Apague a tarefa com ID 123."

Conclusão: Conversa e Cardápio

Lembre-se sempre da analogia:

  • HTTP é o idioma e as regras da conversa (o garçom).
  • A API é o cardápio que lista tudo o que você pode pedir.

Com esses dois conceitos, você tem a base para entender como praticamente todos os sistemas na internet trocam informações, a base para qualquer automação ou integração.
