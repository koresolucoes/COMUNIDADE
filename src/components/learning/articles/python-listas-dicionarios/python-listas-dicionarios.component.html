Estruturas de Dados: Listas e Dicionários

Aprenda a organizar dados com as duas estruturas mais importantes do Python: listas para coleções ordenadas e dicionários para pares chave-valor.

Por que Variáveis Simples não Bastam?

Imagine que você precisa guardar o nome de 100 clientes. Criar cliente1, cliente2, ..., cliente100 seria impossível de gerenciar.

Para lidar com coleções de dados e informações complexas, o Python nos dá duas "super-caixas": Listas e Dicionários. Entendê-las é fundamental, pois é assim que dados de planilhas, APIs e bancos de dados são representados no código.

format_list_numbered 1. Listas (Lists): A Ordem Importa

Uma lista é uma coleção ordenada de itens. É como uma estante numerada. O segredo é que a contagem em programação sempre começa no zero.

  • Usamos colchetes [] para criar listas.
  • Usamos o índice (número) para acessar um item: lista[0].
  • Listas são mutáveis: podemos adicionar, remover ou mudar itens.

vpn_key 2. Dicionários (Dicts): Chave e Valor

Uma lista é ruim se você quiser encontrar o "email do usuário", pois você teria que saber em qual número (índice) o email está.

Um Dicionário resolve isso usando Chaves (nomes) em vez de números. É a estrutura perfeita para representar um objeto do mundo real (uma pessoa, um produto, um pedido). É equivalente ao JSON.

  • Usamos chaves {{ '{' }} {{ '}' }} para criar dicionários.
  • Acessamos o valor usando a chave: dicionario["chave"].

3. O Poder Real: Listas dentro de Dicionários

No mundo real (como no n8n ou em APIs), os dados vêm misturados. Você terá uma Lista de Dicionários (vários usuários) ou um Dicionário contendo uma Lista (um usuário com vários telefones).

Desafio Final

Tente acessar um dado específico dentro de uma estrutura aninhada. Lembre-se: se é lista, use [numero]. Se é dicionário, use ["nome"].
