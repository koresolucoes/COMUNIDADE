Estruturas de Dados: Listas e Dicionários

Aprenda a organizar dados com as duas estruturas mais importantes do Python: listas para coleções ordenadas e dicionários para pares chave-valor.

Os tipos primitivos (strings, números, booleanos) são ótimos, mas raramente trabalhamos com um dado isolado. Geralmente, temos coleções de dados. Python oferece duas estruturas incrivelmente poderosas para isso: Listas e Dicionários. Dominá-las é fundamental para trabalhar com dados.

1. Listas (`list`): A Estante Organizada

Uma lista é uma coleção ordenada e mutável de itens. Pense nela como uma estante com prateleiras numeradas, começando do zero. Você pode colocar qualquer tipo de dado em cada prateleira e acessá-lo pelo número (o índice). Listas são definidas por colchetes [].

Exemplo: Uma lista de tarefas

tarefas = ["Estudar Python", "Limpar dados", "Criar visualização"]

# Acessando o primeiro item (índice 0)
primeira_tarefa = tarefas[0]
print(f"Primeira tarefa: {{'{'}}primeira_tarefa{{'}'}}")  # Saída: Primeira tarefa: Estudar Python

# Acessando o último item
ultima_tarefa = tarefas[-1]
print(f"Última tarefa: {{'{'}}ultima_tarefa{{'}'}}") # Saída: Última tarefa: Criar visualização

# Adicionando um novo item ao final da lista
tarefas.append("Apresentar resultados")
print(f"Lista atualizada: {{'{'}}tarefas{{'}'}}")

# Verificando o tamanho da lista
print(f"Total de tarefas: {{'{'}}len(tarefas){{'}'}}") # Saída: Total de tarefas: 4

2. Dicionários (`dict`): O Arquivo Etiquetado

Um dicionário é uma coleção não ordenada de pares chave-valor. Pense nele como um arquivo de fichas, onde cada ficha tem uma etiqueta (a chave) e um conteúdo (o valor). Chaves são únicas e geralmente são strings. Dicionários são definidos por chaves {{'{'}}{{ '}' }}.

Esta estrutura é a representação em Python do formato JSON que vimos anteriormente.

Exemplo: Descrevendo um usuário

usuario = {{ '{' }}
    "nome": "Ana Clara",
    "email": "ana.clara@example.com",
    "idade": 28,
    "ativa": True
{{ '}' }}

# Acessando um valor pela sua chave
email_do_usuario = usuario["email"]
print(f"O email é: {{'{'}}email_do_usuario{{'}'}}")

# Adicionando um novo par chave-valor
usuario["cidade"] = "São Paulo"
print(f"Usuário atualizado: {{'{'}}usuario{{'}'}}")

# Modificando um valor existente
usuario["ativa"] = False
print(f"Status da conta: {{'{'}}usuario['ativa']{{'}'}}")

Combinando Listas e Dicionários: O Poder Real

A verdadeira mágica acontece quando combinamos essas duas estruturas. Isso nos permite representar praticamente qualquer tipo de dado do mundo real, como uma lista de clientes, onde cada cliente é um dicionário de informações.

Exemplo: Lista de Usuários

usuarios = [
    {{ '{' }}
        "nome": "Ana Clara",
        "email": "ana.clara@example.com",
        "interesses": ["Python", "Ciência de Dados"]
    {{ '}' }},
    {{ '{' }}
        "nome": "Bruno Costa",
        "email": "bruno.costa@example.com",
        "interesses": ["Automação", "APIs", "n8n"]
    {{ '}' }}
]

# Para pegar o email do segundo usuário (índice 1):
email_bruno = usuarios[1]["email"]
print(f"Email do Bruno: {{'{'}}email_bruno{{'}'}}")

# Para pegar o primeiro interesse da Ana (índice 0):
interesse_ana = usuarios[0]["interesses"][0]
print(f"Principal interesse da Ana: {{'{'}}interesse_ana{{'}'}}")
