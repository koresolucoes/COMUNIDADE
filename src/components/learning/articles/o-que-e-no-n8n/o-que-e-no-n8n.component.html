O que é um "Nó" (Node)?

Entenda os blocos de construção de qualquer workflow: os Nós. Aprenda a diferença crucial entre um nó de Gatilho e um nó de Ação.

A Analogia dos Blocos de Construção

Se um workflow no n8n é um fluxograma, então os Nós (Nodes) são os blocos individuais que compõem esse fluxograma. Cada nó representa uma única unidade de trabalho, uma ação específica ou um ponto de partida.

Você constrói sua automação arrastando esses blocos para o Canvas e conectando-os para definir a ordem em que as ações devem acontecer.

A Anatomia de um Nó

Visualmente, um nó é uma caixa que representa uma aplicação ou uma função. Ele tem pontos de entrada (à esquerda) e pontos de saída (à direita) para conectar com outros nós.

Quando você seleciona um nó, o Painel de Parâmetros se abre, permitindo que você configure exatamente o que aquele nó deve fazer.

Os Dois Tipos Fundamentais de Nós

Todos os nós no n8n se encaixam em duas categorias principais, e entender essa diferença é crucial.

boltNós de Gatilho (Triggers)

Um nó de gatilho é sempre o primeiro nó de um workflow. Ele é o ponto de partida, o evento que "dispara" a automação. Um workflow só pode ter um gatilho ativo por vez.

Exemplos: Webhook (começa quando recebe uma notificação), Cron (começa em um horário agendado), Form Trigger (começa quando um formulário é enviado).

play_arrowNós de Ação (Regular Nodes)

Estes são todos os outros nós. Eles executam uma tarefa: buscar dados, enviar dados, manipular informações, tomar decisões, etc. Eles sempre vêm depois de um gatilho ou de outro nó de ação.

Exemplos: HTTP Request (conecta a uma API), Google Sheets (lê ou escreve em uma planilha), Set (cria ou modifica dados), IF (toma uma decisão).

Conclusão: O Coração da Automação

Entender que todo workflow começa com um Gatilho e é seguido por uma ou mais Ações é a base para pensar em automações no n8n. Os nós são as peças do quebra-cabeça; saber como e quando usar cada tipo é o que permite construir fluxos de trabalho poderosos.
