Conectando ao Mundo: O Nó "HTTP Request"

Aprenda a buscar dados de APIs externas ou enviar informações para outros sistemas usando o nó HTTP Request.

O Nó Mais Poderoso do n8n

O n8n tem centenas de nós para serviços específicos (Google Sheets, Discord, etc.). Mas e se você precisar se conectar a um serviço que não tem um nó dedicado? É aí que entra o HTTP Request Node.

Este nó é um "canivete suíço" que permite que você se comunique com praticamente qualquer API na internet. Ele permite que você construa e envie requisições HTTP customizadas, dando a você um poder quase ilimitado de integração.

Configurando uma Requisição GET

Uma requisição GET é usada para buscar dados. Vamos ver os campos principais para configurar uma.

Exemplo: Buscando um usuário de uma API de teste

  • URL: O endereço do endpoint da API.
    https://jsonplaceholder.typicode.com/users/1
  • Method: O verbo HTTP.
    GET
  • Response Format: Como o n8n deve interpretar a resposta.
    JSON

Ao executar este nó, o n8n fará a requisição e o JSON da resposta estará disponível em $json para o próximo nó.

Enviando Dados com POST

Uma requisição POST é usada para criar um novo recurso. A configuração é similar, mas com a adição do "corpo" (Body) da requisição.

Exemplo: Criando um novo post

  • URL: https://jsonplaceholder.typicode.com/posts
  • Method: POST
  • Body Content Type: JSON
  • Body: Aqui você insere o JSON que quer enviar. Você pode usar expressões para torná-lo dinâmico! 
    {{ '{' }}
  "title": "{{ '{{' }} $json.titulo_do_passo_anterior {{ '}}' }}",
  "body": "Este é o conteúdo do post.",
  "userId": 1
{{ '}' }}

Autenticação com Cabeçalhos (Headers)

A maioria das APIs exige algum tipo de autenticação. A forma mais comum é através de um Cabeçalho (Header). No nó HTTP Request, você pode adicionar headers na seção "Authentication" ou diretamente na aba "Headers".

Exemplo: Enviando uma Chave de API

  • Na aba "Headers", clique em "Add Header".
  • Name: X-Api-Key (ou o nome que a API exigir, como Authorization).
  • Value: SUA_CHAVE_DE_API_SECRETA
Atenção: Nunca coloque chaves de API diretamente no campo "Value"! O próximo artigo ensinará a forma correta e segura de fazer isso usando Credenciais.

Conclusão: A Porta para o Mundo

O nó HTTP Request é a sua porta de entrada para se conectar com qualquer serviço que tenha uma API. Dominá-lo significa que você não está mais limitado aos nós pré-construídos do n8n; você pode integrar qualquer ferramenta que precise, tornando suas possibilidades de automação praticamente infinitas.
