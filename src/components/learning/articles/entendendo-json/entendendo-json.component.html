Entendendo JSON: A Estrutura de Dados da Web

Aprenda a ler e entender o formato JSON, a maneira mais comum de trocar dados entre sistemas na internet.

Quando uma API nos responde ou quando um webhook nos envia dados, essa informação precisa vir em um formato estruturado que tanto humanos quanto máquinas possam entender. O formato mais popular para isso é o JSON (JavaScript Object Notation).

A Analogia do Cartão de Visitas

Pense em um cartão de visitas. Ele contém informações estruturadas:

Nome: João Silva

Cargo: Desenvolvedor

Telefone: (11) 99999-8888

Ativo: Sim

Cada informação tem uma "etiqueta" (Nome, Cargo) e um "valor" (João Silva, Desenvolvedor). O JSON funciona exatamente da mesma maneira, mas com uma sintaxe específica que os computadores entendem.

O que é JSON?

JSON é um formato de texto para representar dados estruturados. É "a língua que os dados falam na web". Vamos traduzir nosso cartão de visitas para JSON:

Cartão de Visitas em JSON

{{ '{' }}
  "nome": "João Silva",
  "cargo": "Desenvolvedor",
  "telefone": "(11) 99999-8888",
  "ativo": true
{{ '}' }}

As Regras da Sintaxe

O JSON tem poucas e simples regras:

  • Os dados são organizados em pares de "chave": valor.
  • As chaves devem ser sempre texto, envoltas em aspas duplas ".
  • Cada par chave-valor é separado por dois pontos :.
  • Pares são separados uns dos outros por vírgulas ,.
  • Objetos (como nosso cartão) são envoltos por chaves {{ '{' }} {{ '}' }}.
  • Arrays (listas ordenadas) são envoltos por colchetes [ ].

Os Tipos de Dados em JSON

Os "valores" em JSON podem ser de diferentes tipos, muitos dos quais você já conhece:

{ "exemplo_string": "Olá, mundo!" } // Texto, sempre com aspas duplas.

{ "exemplo_numero": 123.45 } // Números (inteiros ou decimais), sem aspas.

{ "exemplo_booleano": true } // Verdadeiro ou falso (true ou false), sem aspas.

{ "exemplo_array": [ "maçã", "banana", "laranja" ] } // Uma lista de valores.

{ "exemplo_objeto": {{ '{' }} "chave": "valor" {{ '}' }} } // Outro objeto JSON aninhado.

{ "exemplo_nulo": null } // Representa a ausência de valor.

Juntando Tudo: Um Exemplo Complexo

Vamos ver como o JSON pode descrever algo mais complexo, como um produto em um site de e-commerce. Note como podemos combinar objetos e arrays para criar estruturas ricas.

Exemplo: JSON de um Produto

{{ '{' }}
  "id": 101,
  "nome": "n8n Workflow Automation Tool",
  "disponivel": true,
  "preco": null,
  "caracteristicas": [
    "Código Aberto",
    "Auto-hospedável",
    "Extensível"
  ],
  "conexoes_populares": {{ '{' }}
    "armazenamento": "Google Drive",
    "comunicacao": "Discord",
    "crm": "HubSpot"
  {{ '}' }}
{{ '}' }}

Conclusão: Simples, Leve e Universal

O JSON se tornou o padrão da web por três motivos principais: é leve (ocupa pouco espaço), é fácil para humanos lerem e é fácil para máquinas interpretarem. Ao entender sua estrutura, você está um passo mais perto de dominar como os sistemas se comunicam e como manipular os dados que fluem entre eles.
