Manipulando Arquivos: Dados Binários

Entenda como o n8n lida com arquivos (imagens, PDFs) e como ler, criar e enviar dados binários em seus workflows.

Além do JSON: O Mundo dos Arquivos

Até agora, lidamos principalmente com dados em formato JSON. Mas muitas automações envolvem arquivos: ler um PDF, baixar uma imagem, enviar um anexo por e-mail, etc.

O n8n trata arquivos como dados binários. Enquanto o JSON é texto estruturado, um dado binário é a "matéria-prima" de um arquivo, uma sequência de bytes que o computador entende.

A Estrutura de Dados: `json` e `binary`

Quando um nó lida com um arquivo, a estrutura de dados de um item no n8n ganha um novo protagonista. Além da propriedade json, agora temos a propriedade binary.

Exemplo: Saída de um nó "Read Binary File"

[
  {{ '{' }}
    "json": {{ '{' }}
      // Metadados sobre o arquivo podem aparecer aqui
      "fileName": "relatorio.pdf",
      "mimeType": "application/pdf"
    {{ '}' }},
    "binary": {{ '{' }}
      // A propriedade que contém os dados do arquivo
      "data": {{ '{' }}
        "type": "Buffer",
        "data": [ 80, 68, 70, 45, ... ]
      {{ '}' }}
    {{ '}' }}
  {{ '}' }}
]

O importante é saber que quando um arquivo está "anexado" a um item, ele vive dentro da propriedade binary.

Como Usar Dados Binários

A maioria dos nós que trabalham com arquivos sabe procurar por essa propriedade binary automaticamente.

  • Enviar um anexo: Se você conectar um nó que leu um arquivo a um nó "Send Email", o nó de e-mail automaticamente pegará o dado binário e o adicionará como anexo.
  • Fazer upload para um serviço: Se você conectar a um nó "Google Drive", ele fará o upload do arquivo contido no binary.
  • Enviar em uma requisição HTTP: No nó "HTTP Request", você pode especificar que o corpo da requisição deve usar o "Binary Data", e o n8n enviará o arquivo.

Conclusão: O "Pacote Físico" da Automação

Se os dados JSON são as "cartas e documentos" que fluem pelo seu workflow, os dados binários são os "pacotes físicos". O n8n gerencia ambos de forma transparente, permitindo que você crie automações que não apenas manipulam informações, mas também leem, criam e movem arquivos entre diferentes sistemas.
