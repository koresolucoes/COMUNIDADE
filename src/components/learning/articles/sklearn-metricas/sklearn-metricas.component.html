Avaliando a Performance do seu Modelo

Aprenda a usar métricas como Acurácia e Erro Quadrático Médio para saber se seu modelo está performando bem.

Por que Avaliar? A Prova dos Nove

No artigo anterior, treinamos um modelo e vimos que suas previsões pareciam "próximas" dos valores reais. Mas em Machine Learning, "próximo" não é suficiente. Precisamos de números objetivos — métricas — para quantificar o quão bom (ou ruim) nosso modelo realmente é.

Avaliar o modelo nos permite comparar diferentes algoritmos, ajustar parâmetros e, o mais importante, saber se podemos confiar nas suas previsões no mundo real.

Métricas de Regressão: Medindo o Erro

Para problemas de regressão (prever um número), nossas métricas geralmente medem o "erro" ou a distância entre a previsão e o valor real. O Scikit-learn nos oferece várias, e vamos focar em três das mais importantes.

1. Erro Absoluto Médio (MAE - Mean Absolute Error)

O que é: A média da diferença absoluta (sem sinal de +/-) entre cada previsão e seu valor real.

Como interpretar: "Em média, meu modelo erra por X unidades". É fácil de entender porque está na mesma unidade do que estamos prevendo (ex: dólares, anos, etc.). Um MAE menor é melhor.

2. Erro Quadrático Médio (MSE - Mean Squared Error)

O que é: Similar ao MAE, mas eleva ao quadrado cada erro antes de tirar a média. A Raiz do Erro Quadrático Médio (RMSE) é simplesmente a raiz quadrada do MSE, o que a traz de volta para a unidade original.

Como interpretar: O MSE penaliza erros grandes muito mais do que erros pequenos. Se um erro de $1000 é muito pior do que dois erros de $500, o MSE é uma boa métrica. Um MSE/RMSE menor é melhor.

3. Coeficiente de Determinação (R²)

O que é: Uma métrica que vai de 0 a 1 (ou até negativa para modelos muito ruins) e representa a proporção da variância na variável alvo que é previsível a partir das features.

Como interpretar: "Meu modelo consegue explicar X% da variação nos preços das casas". Um R² de 0.60 significa que o modelo explica 60% da variação. Um R² mais próximo de 1 é melhor.

Calculando as Métricas no Código

Vamos continuar de onde paramos no artigo anterior e calcular essas métricas para as previsões que fizemos.

Exemplo: Avaliando nosso modelo de preços de casas

# Suponha que as variáveis X_train, X_test, y_train, y_test, model e predictions já existem
from sklearn.metrics import mean_absolute_error, mean_squared_error, r2_score
import numpy as np # Usaremos numpy para a raiz quadrada

# 1. Erro Absoluto Médio (MAE)
mae = mean_absolute_error(y_test, predictions)
print(f"MAE: {{'{'}}mae:.2f{{'}'}}")
# Interpretação: Em média, nosso modelo erra em $53,300 (o target está em unidades de $100,000)

# 2. Erro Quadrático Médio (MSE) e sua Raiz (RMSE)
mse = mean_squared_error(y_test, predictions)
rmse = np.sqrt(mse)
print(f"MSE: {{'{'}}mse:.2f{{'}'}}")
print(f"RMSE: {{'{'}}rmse:.2f{{'}'}}")
# Interpretação: O RMSE é maior que o MAE, indicando que temos alguns erros grandes.

# 3. R-quadrado (R²)
r2 = r2_score(y_test, predictions)
print(f"R²: {{'{'}}r2:.2f{{'}'}}")
# Interpretação: Nosso modelo simples explica cerca de 60% da variação nos preços das casas.

Conclusão: Quantificando o Sucesso

Agora temos uma base objetiva para julgar nosso modelo. Um R² de 0.60 não é fantástico, mas para um modelo simples sem nenhum tratamento de dados, é um começo razoável.

Nosso próximo passo será melhorar esse número. Para isso, precisamos olhar para os nossos dados de entrada e prepará-los melhor para o modelo, um processo chamado Feature Engineering.
